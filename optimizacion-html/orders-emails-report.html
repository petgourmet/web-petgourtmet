📦 Actualización del Centro de Logística

Mejoras de UI, rediseño de correos electrónicos corporativos y nuevos estados de pedidos (En camino / Reembolsado).

1. Interfaz de Administración de Pedidos

  • ✅

    Nuevos estados soportados Nuevo

    Se añadieron los estados "En camino" (shipped) y "Reembolsado" (refunded) en todas las listas de selección, filtros y reportes.

  • ✅

    Nuevos Badges visuales

    Se definieron badges con colores específicos e iconos para identificar fácilmente cuando un envío se encuentra en camino (🚚 Indigo) o ha sido reembolsado (💸 Naranja).

  • ✅

    Ampliación de contededor Horizontal

    Se ajustaron los márgenes y estructuración visual en la página de pedidos para maximizar el espacio horizontal y mostrar la información de forma más clara en resoluciones Desktop.

2. Correos Electrónicos (Email Service)

  • 📧

    Rediseño premium con branding corporativo

    El template HTML de los correos ha sido rediseñado utilizando un estilo en bloque estético #7AB8BF, incorporando el logo en formato elegante. La tabla de contenido ahora posee mucho más "aire" e indentación profesional para alinearse con la identidad de marca de Pet Gourmet.

  • 📧

    Plantillas para los nuevos estados de envío

    El sistema ahora sabe distinguir entre órdenes completadas, procesadas, enviadas a la paquetería y canceladas/reembolsadas, cambiando automáticamente el asunto ("¡Tu pedido va en camino!" / "Reembolso procesado") y el texto para el cliente final.

  • 🧾

    Información Adicional Detallada

    La notificación por correo ahora integra el resumen logístico de envío mostrando el domicilio de entrega seleccionado por el cliente, además del detalle del pago (costo del envío, impuestos, subtotal). Esto transmite mayor tranquilidad.

3. API & Funciones Core

  • ⚙️

    Actualización tipo-seguridad

    Se modificaron las validaciones de Typescript en el Route Handler /api/admin/update-order-status para admitir correctamente strings 'shipped' y 'refunded'.

  • ⚙️

    Autodetección

    El código previene enviar plantillas equivocadas y avisa al sistema si se lanza una actualización de estado de orden que no es válida dentro del arreglo de validStatuses.
