🔧 Diagnósticos Adicionales (Lighthouse)

🔄
20 long tasks encontradas — 8.9s main-thread work
Después de aplicar las Fases 1–3, las long tasks caerán dramáticamente porque: (1) YouTube iframe deja de cargar en el critical path, (2) Framer Motion se carga lazy, (3) los trackers no compiten con el render. Las long tasks que queden pueden atacarse con scheduler.yield() o dividiendo procesamiento en chunks (React 19 concurrency).
👂
Passive listeners — scroll sin {passive: true}
Lighthouse detecta event listeners de scroll sin { passive: true }. Revisar components/particles-background.tsx y cualquier window.addEventListener('scroll', ...) o document.addEventListener('touchstart', ...). Añadir el flag passive: true. Ejemplo: window.addEventListener('scroll', handler, {'{'} passive: true {'}'})
📦
Cache policy — 4 recursos sin cache headers (Netlify)
En netlify.toml o vercel.json, añadir headers de cache para assets estáticos: Cache-Control: public, max-age=31536000, immutable para /public/* con hash en el nombre. Next.js ya hace esto para el directorio _next/static/, pero archivos en /public/ no.
🎮
Non-composited animations — 2 elementos
Lighthouse detecta 2 animaciones que no usan GPU-composited layers. Revisar animaciones CSS que usen width, height, top, left en lugar de transform y opacity. Los hover effects de las categorías en app/page.tsx (transition-all) posiblemente son los culpables. Cambiar a transition-transform y transition-opacity.
