Mejora 1 – Campos adicionales en formularios de checkout

Archivos modificados

  • components/checkout-modal.tsx Modal overlay de checkout – campo dirección 2 + selector código de país
  • app/checkout/page.tsx Página completa de checkout – mismos campos nuevos
  • lib/stripe/checkout-service.ts Interfaz ShippingInfo – address2?: string
  • lib/email-service.ts Plantillas HTML de email – renderiza address2, fix postalCode

Detalle de campos añadidos

Campo «Dirección 2» (address2): campo opcional para número de interior, piso, departamento, etc. Se añadió en ambos formularios (modal y página completa) y se transmite al servicio de checkout de Stripe y a las plantillas de email de confirmación.

Selector de código de país: dropdown con 10 países de LATAM + España (default +52 México) precediendo al campo de teléfono. Carga automáticamente el código guardado en el perfil del usuario.

Mejoras 2, 3 y 4 – Botones de compra directa

Flujo anterior (con carrito)

Click en "Comprar"
→
addToCart()
→
Modal carrito se abre ❌
→
Usuario cierra el modal
→
Usuario navega a checkout manualmente

Flujo nuevo (checkout directo)

Click en "Comprar"
→
addToCart()
→
setShowCart(false) ✓
→
router.push('/checkout') ✓

Causa raíz del problema

La función addToCart() en components/cart-context.tsx siempre ejecuta setShowCart(true) internamente (línea 144). No podía modificarse esa lógica global porque otros flujos (navbar, catálogo) sí necesitan abrir el carrito. La solución fue capturar setShowCart del contexto en la página de producto y llamarlo con false inmediatamente después de que handleAddToCart() retorna true.

Cambios en app/producto/[slug]/page.tsx

Antes

// Solo destructuraba addToCart
const { addToCart } = useCart()
const router = useRouter()

// Botón "Comprar ahora"
onClick={() => {
  setIsSubscription(false)
  if (!product?.subscription_available) {
    if (handleAddToCart()) {
      router.push('/checkout')
    }
    // ❌ carrito se abre igual
  }
}}

// Botón "Comprar"
onClick={handleAddToCart}
// ❌ no navega, abre carrito

Después

// Captura también setShowCart
const { addToCart, setShowCart } = useCart()
const router = useRouter()

// Botón "Comprar ahora"
onClick={() => {
  setIsSubscription(false)
  if (!product?.subscription_available) {
    if (handleAddToCart()) {
      setShowCart(false) // ✅
      router.push('/checkout')
    }
  }
}}

// Botón "Comprar"
onClick={() => {
  if (handleAddToCart()) {
    setShowCart(false) // ✅
    router.push('/checkout')
  }
}}

Arquitectura relevante

Contexto del carrito (components/cart-context.tsx)

// Las funciones expuestas por el contexto
interface CartContextType {
  cart: CartItem[]
  addToCart: (item: CartItem) => void   // siempre setShowCart(true)
  removeFromCart: (index: number) => void
  showCart: boolean
  setShowCart: (show: boolean) => void  // ← clave para suprimir
  showCheckout: boolean
  setShowCheckout: (show: boolean) => void
}

handleAddToCart en página de producto

const handleAddToCart = (): boolean => {
  // Validación de variante si producto variable
  if (isVariableProduct && !selectedVariant) {
    toast({ title: "Selecciona una variante" })
    return false           // ← retorna false si falla
  }
  addToCart({ ... })
  return true             // ← retorna true si se añadió ok
}

// Los botones usan el booleano para decidir si navegar:
if (handleAddToCart()) {
  setShowCart(false)    // cancela apertura del sidebar
  router.push('/checkout')
}
